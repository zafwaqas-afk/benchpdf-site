What comes through

  • The reading order of the page, so sentences arrive in the order they were written.
  • Line and paragraph breaks, as closely as the PDF's own structure allows.
  • Page breaks, marked with a form feed character between pages.
  • The characters themselves as UTF-8, so accents and symbols survive.

Where it stops

Plain text has no concept of columns, so a two-column layout or a table is flattened into a single run of text. It stays readable but the shape is gone. A scanned PDF has no text underneath at all and comes back empty or nearly so, because this reads existing text rather than recognising letters in a picture. That needs OCR, which BenchPDF does not do.

How to convert PDF to text

  • Drop your PDF into the box, or choose a file.
  • Wait a moment while the text is read from each page.
  • Click Download to save the .txt file.
  • Open it in any text editor or word processor.

Does it keep paragraphs and line breaks?

As closely as the PDF allows. Line and paragraph breaks are preserved where the document's own structure records them, and each page is separated by a form feed character. What does not survive is anything positional, so columns and tables are flattened into one reading order.

Why is my file empty?

Almost always because the PDF is a scan. A scanned page is a photograph, with no text underneath for the tool to read. Try selecting a line of text in a PDF viewer: if nothing highlights, there is nothing to extract, and you will need OCR software to recognise the words first.

Does my file get uploaded?

No. The text is read out of the PDF by your own browser and the .txt file is generated on your device. Nothing is sent anywhere. You can watch your browser's network activity during a conversion to confirm it.

What encoding is the output?

UTF-8, so accented characters, currency symbols and other non-ASCII text come through intact rather than as question marks. The file opens in any text editor, word processor or code editor without needing a particular setting.

Is there a length limit?

None is set by the tool. A one page letter extracts in well under a second. A very long report takes a little longer, and the practical ceiling is your own device's memory rather than any cap imposed here, because the work happens locally.

Does it work on a Mac?

Yes. This page runs in the browser, so it works the same on macOS, Windows, Linux and ChromeOS. The separate BenchPDF desktop app is Windows only, but this conversion does not need it.
