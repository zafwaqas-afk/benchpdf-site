How to Convert a Scanned PDF That's Just Images

If you can't select or search the text in a PDF, you're not looking at a document, you're looking at a picture of one. Here's what that means and what you can actually do about it.

What does it mean for a PDF to be “just images”?

It means the PDF is a container for a picture of a page, not for the words on it. Somewhere along the way, whether from a flatbed scanner, a phone camera, or a fax machine, the document was captured as a photograph and placed inside a PDF file. There is no hidden layer of text underneath for a computer to read, only pixels.

This is different from a PDF exported from Word, a web page, or most other software, which stores actual characters, fonts, and positions as data. Two PDFs can look identical on screen while being fundamentally different files underneath: one has real text, the other is a photograph with a .pdf extension.

How can I tell if my PDF is a scan?

Open it in any PDF viewer and try to click and drag across a line of text, the way you would select a sentence to copy it. If a box or highlight appears around the words, there is a real text layer. If nothing happens, or your cursor just draws a rectangle over the image, you are looking at a scan.

Another quick test: use your viewer's search function, usually Ctrl+F, and search for a word you can see on the page. A text-based PDF will find and highlight it. A scanned PDF will report no matches, because there is nothing to search.

Why can't PDF to Word or PDF to Text tools read a scanned PDF?

Every text-extraction tool, BenchPDF included, works by reading the text layer that is already embedded in the PDF. It does not look at the page and recognise letters the way a person does, it reads structured data that says this word, in this font, at this position. A scanned PDF has none of that data to read.

This is not a limitation specific to any one converter. Whether you use BenchPDF, Word's own PDF import, or any other tool built the same way, a scanned page will come back empty, because the underlying problem is the file itself, not the software reading it.

What does BenchPDF do with a scanned PDF?

It is honest about the limitation rather than guessing. PDF to Text returns an empty or near-empty result for a scanned page, since there is genuinely no text to extract. PDF to PowerPoint detects that a page has little or no extractable text and falls back to placing it as a full-slide image instead of an empty slide, so nothing is visually lost.

PDF to Images sidesteps the whole issue, since it does not try to read text at all, it renders each page exactly as it looks, pixel for pixel. That makes it the most reliable of BenchPDF's tools for a scanned document, if a picture of the page is what you actually need.

Can I still get something useful out of a scanned PDF?

Yes, depending on what you need. If you just need to view, print, share, or embed the pages somewhere, converting to images or leaving it as a PDF works fine, since nothing about the visual content is lost. If you need the actual words as editable or searchable text, that requires a different kind of tool entirely, described below.

What about a PDF that's a mix of scanned and text pages?

This is common: a cover sheet or a signed page scanned in and stapled, in file terms, to the front of an otherwise normal, text-based document. BenchPDF handles each page independently, so this works out better than you might expect. PDF to Text will return real text for the typed pages and nothing for the scanned one. PDF to PowerPoint will build editable text boxes for the typed pages and place the scanned page as a single image, page by page, rather than treating the whole file as one failure.

The practical effect is that you do not need to separate a mixed document before converting it, the per-page detection already does that work. If the scanned page specifically needs its text recognised, that one page is the only part you would need to run through OCR software separately.

Does BenchPDF do OCR?

No, not currently. OCR, optical character recognition, is the specific technology that looks at an image of text and works out what the letters actually are, effectively teaching a computer to read a photograph. BenchPDF's conversions rely on text that is already present in the PDF; recognising text inside an image is a separate, more involved problem that is not part of the app today.

What are my options if I need the text recognised?

You will need dedicated OCR software, which exists as a distinct category of tool built specifically for this. Many document scanning apps, and some PDF viewers, include an OCR step at the point of scanning, which is often the most reliable time to do it, since the software works from the same clean image the scanner produced rather than a recompressed copy.

If you scan documents regularly, it is worth checking whether the software you already use, your scanner's own app, or your PDF reader, has an OCR option built in before looking for a separate tool. Once a PDF has been through OCR and has a real text layer, BenchPDF's own conversions, including PDF to Word and PDF to PowerPoint, work on it normally.

  • PDF to Text, for text-based PDFs, extracts the words directly with no layout.
